Bottom line.

Out of the box, with default settings, Redis 8.6 is faster. It beats Valkey 9.1 by 3-6% in most workloads, and wins read-heavy tests by 5%. In the high-throughput pipeline test, the two are neck and neck at 1.95M req/s.

The picture changes when you turn on io-threads. Valkey 9.1 jumps to 2.54M req/s, 15.5% ahead of Redis 8.6. Write-heavy workloads show an even bigger gap at 11.2%. The reason: Valkey gets 30% more throughput from io-threads, while Redis 8.6 gets 13%. Redis 7.2 gets almost nothing.

Redis 8.6 still has an edge in tail latency (p99.9: 1.43ms vs 1.67ms) and read-only throughput (+2.8%). But for most production deployments running io-threads, Valkey 9.1 is the faster engine. It's also BSD-3 licensed.
